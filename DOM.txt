1.事件委托
答: 

dom中事件的触发有两个部分: 事件捕获与事件冒泡
(1) 事件捕获
当我点击了一个ul中的li中的div时，此时点击事件会从最外层
向里面传递: html -> body -> ul -> li -> div ，所有途径的
元素都会有这个点击事件
(2) 事件冒泡
与事件捕获相反，当我点击了一个ul中的li中的div时，此时点击事件
会从最里层向外面传递: div -> li -> ul -> body -> html ，
所有途径的元素都会有这个点击事件
(3) 事件委托
事件委托就是基于事件捕获和事件冒泡，将要触发的事件放在父亲
元素上，有两点好处: 一是节约代码优化性能，不用在所有的孩子
上都添加点击事件；二是可以监听到未来新生成的孩子的点击事件

Dom事件模型: 对每个事件先捕获再冒泡
阻止事件冒泡: e.stopPropagation()   e.preventDefault阻止默认行为
事件绑定时都是默认绑定的事件冒泡，想要事件捕获得加上true

ul.addEventListener('click', (e) => {
            if(e.target.className.toLowerCase() === 'btn2') {
                alert('ul被点击了')
            }   
        })
// 如果被点击的是ul中的btn2，ul上的点击事件才触发
